# Glossary

- **OAuth 2.0**: Authorization framework allowing apps to obtain delegated access tokens.
- **Authorization Code**: Short-lived code returned from `/oauth2/authorize`, exchanged at `/oauth2/token`.
- **PKCE**: OAuth extension for public clients using a verifier/challenge proof.
- **code_verifier**: Random secret generated by client and sent only to token endpoint.
- **code_challenge**: Hash-derived value from verifier sent to authorize endpoint.
- **Access Token**: Token used to call protected APIs.
- **Refresh Token**: Long-lived token used to obtain new access tokens.
- **Redirect URI**: Callback URI registered in OAuth client (here: `myapp://callback`).
- **Deep Linking**: Technique for returning from browser to native app via URI scheme.
- **Client ID**: Public identifier of OAuth application in FusionAuth.
- **State**: Random anti-CSRF value sent in authorize request and validated on callback.
